# 专家会诊系统快速开始指南

## 第一步：准备医疗数据

确保您已经使用 `/save-report` 命令保存了医疗检查单数据。

如果没有数据，可以使用示例数据测试：
```bash
# 示例数据位于：医疗报告/示例数据-生化检查.json
```

## 第二步：启动多学科专家会诊

### 基础用法

```bash
# 会诊最近3条记录（默认）
/consult

# 会诊所有数据
/consult all

# 会诊最近5条记录
/consult recent 5

# 会诊指定日期
/consult date 2025-12-31
```

### 系统会做什么？

1. **自动识别专科**
   - 系统会分析您的检查数据
   - 自动识别需要哪些专科参与会诊
   - 例如：血脂异常 → 心内科，血糖异常 → 内分泌科

2. **并行启动专家**
   - 同时启动多个专科的 subagent
   - 每个专家独立分析数据
   - 提高分析效率

3. **生成综合报告**
   - 汇总各专科意见
   - 按优先级排序问题
   - 提供综合管理建议
   - 制定随访计划

## 第三步：查看会诊报告

会诊报告包含：

### 🔴 紧急问题
需要立即就医的情况

### 🟡 重要问题
需要近期处理的问题

### 🟢 常规问题
可以定期随访的问题

### 各专科详细分析
每个专科的独立分析

### 综合建议
- 生活方式干预
- 检查/复查计划
- 专科会诊建议

## 第四步：单专科深入咨询

如果某个专科的问题特别突出，可以深入咨询：

```bash
# 查看支持的专科
/specialist list

# 咨询心内科
/specialist cardio recent 5

# 咨询内分泌科
/specialist endo all

# 咨询消化科
/specialist gastro recent 3
```

## 完整示例

假设您刚做完体检，有多项异常：

```bash
# 第一步：启动会诊
/consult all

# 第二步：查看报告
# 系统会生成完整的会诊报告

# 第三步：深入咨询
# 如果血脂问题突出，深入咨询心内科
/specialist cardio all
```

## 重要提示

### ⚠️ 安全红线

系统严格遵守以下原则：
- ❌ 不给出具体用药剂量
- ❌ 不直接开具处方药名
- ❌ 不判断生死预后
- ❌ 不替代医生诊断

### ✅ 系统提供的价值

- 解读医疗检验指标的临床意义
- 识别异常指标和潜在风险
- 提供健康生活方式建议
- 推荐针对性检查项目
- 协助制定随访计划
- 整合多学科专家意见

### 📋 使用建议

1. **定期会诊**：每次有新检查结果后运行 `/consult`
2. **专项深入**：对重点异常使用 `/specialist` 深入咨询
3. **趋势观察**：使用 `recent` 参数观察指标变化
4. **理性参考**：报告仅供参考，诊疗需咨询医生

## 常见问题

### Q1: 系统会代替医生吗？
**A:** 不会。系统只提供指标解读和建议，不做诊断，所有诊疗决策需咨询专业医生。

### Q2: 会诊报告准确吗？
**A:** 报告基于AI分析，仅供参考。建议结合医生意见综合判断。

### Q3: 可以直接按报告用药吗？
**A:** 不可以。系统不提供具体药物剂量，用药必须遵医嘱。

### Q4: 多久会诊一次？
**A:** 建议每次有新检查结果后会诊一次，或按医生建议定期随访。

### Q5: 数据安全吗？
**A:** 所有数据存储在本地，不上传云端，完全私有。
